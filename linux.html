Linux Privilege Escalation

Comandos, técnicas y herramientas para la enumeración y escalada de privilegios en Linux.

Enumeración

hostname

Nombre del host de destino.

uname -a

Detalles adicionales sobre el kernel utilizado por el sistema.

cat /proc/version

El sistema de archivos proc (procfs) proporciona información sobre el sistema de destino, incluyendo detalles del kernel.

cat /etc/issue

Suele contener información sobre el sistema operativo. Puede estar personalizado.

ss -tuln

Puertos abiertos en el sistema.

ps [opciones]

Muestra procesos en ejecución. Campos: PID, TTY, Time, CMD.

ps -A Muestra todos los procesos en ejecución.
ps axfjMuestra árbol de procesos.
ps auxProcesos de todos los usuarios, con usuario iniciador y procesos sin terminal.
env

Variables de entorno del sistema.

sudo -l

Lista todos los comandos que el usuario actual puede ejecutar con sudo.

ifconfig

Información sobre las interfaces de red del sistema.

ip route

Rutas de red configuradas.

netstat [opciones]

Información sobre conexiones existentes.

netstat -aTodos los puertos en escucha y conexiones establecidas.
netstat -at / -auListar protocolos TCP o UDP respectivamente.
netstat -lPuertos en modo escucha.
netstat -sEstadísticas de uso de red por protocolo.
netstat -tpConexiones con nombre de servicio e info del PID.
netstat -iEstadísticas de la interfaz.

Comando find

find . -name <name>

Busca el archivo llamado <name> en el directorio actual.

find /<directorio> -name <name>

Busca el archivo en el directorio especificado.

find / -type d -name <nombre>

Busca un directorio con ese nombre en /.

find / -type f -perm 0777

Archivos con permisos 777 — legibles, escribibles y ejecutables por todos los usuarios.

find / -perm a=x

Busca archivos ejecutables.

find /home -user <user>

Todos los archivos del usuario especificado en /home.

find / -mtime <días>

Archivos modificados en los últimos N días.

find / -atime <días>

Archivos accedidos en los últimos N días.

find / -cmin -<minutos>

Archivos modificados en los últimos N minutos.

find / -amin -<minutos>

Archivos accedidos en los últimos N minutos.

find / -size +<tamaño>M

Archivos de ese tamaño o más. Usa - para menor.

find / -writable -type d 2>/dev/null

Carpetas con permisos de escritura para todos los usuarios.

find / -perm -o x -type d 2>/dev/null

Carpetas con permisos de ejecución para todos los usuarios.

find / -perm -u=s -type f 2>/dev/null

Archivos con el bit SUID activado — permite ejecutarlos con privilegios del propietario.

find / -name perl* / -name python* / -name gcc*

Busca intérpretes o compiladores disponibles en el sistema (útil para exploits locales).

SUID / SGID

SUID (Set User ID) y SGID (Set Group ID) permiten ejecutar un archivo con el nivel de permisos del propietario o del grupo, respectivamente.

find / -type f -perm -04000 -ls 2>/dev/null

Lista todos los archivos con los bits SUID o SGID activados.

WEB GTFOBins Binarios Unix que pueden usarse para escalar privilegios ↗

Capabilities

Las Capabilities permiten gestionar privilegios con mayor precisión que SUID, asignando solo los permisos necesarios a un proceso.

getcap -r / 2>/dev/null

Enumera recursivamente todas las capabilities configuradas en el sistema (como usuario sin privilegios).

Cron Jobs

Las tareas cron ejecutan scripts o binarios en momentos específicos. Si una tarea corre como root y el script es modificable, es un vector de escalada.

cat /etc/crontab

Muestra las tareas cron configuradas a nivel del sistema.

PATH Hijacking

Si una carpeta con permisos de escritura está en $PATH, puedes crear un ejecutable falso con el nombre de un comando legítimo para secuestrar su ejecución.

echo $PATH

Muestra el valor actual de la variable PATH.

find / -writable 2>/dev/null | cut -d "/" -f 2,3 | grep -v proc | sort -u

Lista directorios con permisos de escritura, filtrando /proc.

Preguntas clave: ¿Hay carpetas en $PATH con escritura? ¿Se puede modificar $PATH? ¿Existe un script que use comandos sin ruta absoluta?

NFS

Si un recurso NFS está montado con no_root_squash, un cliente remoto puede crear archivos SUID y ejecutarlos en el servidor como root.

cat /etc/exports

Archivo de configuración NFS. Creado durante la instalación del servidor; readable por usuarios normales.

showmount -e <IP>

Lista los recursos NFS exportados del host objetivo.
